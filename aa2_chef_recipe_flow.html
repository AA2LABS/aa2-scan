CHEF RECIPE FLOW — ARCHITECTURE DOCTRINE · PRE-BUILD LOCK
ALLERGY LAYER RULE
Every ingredient in every Chef suggestion runs through The Equalizer against the member's membrane. Not just the scanned item — every item in the full recipe list.
UPGRADE GATE
Non-full members see WITHOUT clearly, WITH blurred. Gate shows what they're missing — allergy layer, full recipe, substitutes. One CTA: Upgrade. No shame.
SECOND CLAUDE CALL
Tapping a recipe chip fires a new Claude API call. Passes: recipe name, member membrane, scanned items. Returns: full ingredient list, HAVE/FLAGGED/NEED split, allergy warnings, safe substitutes, prep note.
COOKBOOK TABLE NEEDED
New Supabase table: cookbook_recipes. Fields: member_id, recipe_name, ingredients (jsonb), membrane_flags (jsonb), created_at, shared (boolean), live_cooking_session_id.
SHOPPING MEMO STATES
DONE (green check), FLAGGED (red warning with allergy note + safe alternates + SCAN TO VERIFY), SCAN NOW (blue bolt), OPEN (empty checkbox). Allergy flags shown inline under the item.
SHARED RECIPE RULE
When shared via Live Cooking, the shared version shows the clean ingredient list only. Membrane flags and personal substitutes are never shared. Personal truth stays private.
ROLE SEPARATION
The Chef suggests recipes, builds ingredient lists, delivers prep notes, assembles cookbook format, suggests alternates. The Equalizer runs every allergy check. These roles never overlap.
ACT RIGHT DOLLARS
Every safe substitute found that saves money vs the flagged product earns Act Right Dollars. The allergy layer is also the savings layer. Both outputs appear on the same flag card.
