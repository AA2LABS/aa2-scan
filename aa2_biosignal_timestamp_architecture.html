ALREADY IN STACK
Why not Whisper, AWS Rekognition, or Azure?
Whisper is pay-per-minute — device-native STT is free and accurate enough for the baseline use case. AWS Rekognition adds emotion percentage confidence scores but costs per API call. For the delta measurement AA2 needs — did the face respond to this stimulus — ML Kit's smile probability is sufficient and costs nothing. Pay-per-use becomes worth it at real-time continuous monitoring scale, which is a future layer.
This is the most critical architectural requirement
Video, voice, and HRV data are only meaningful when they share the same timestamp and are linked to the same Javier stimulus question. The stimulus_id is the binding key. Without it, the comparison engine has no way to know that the smile happened during the stress question — not after it.

How Garmin data gets matched after the fact
Garmin doesn't stream live during onboarding — the member uploads the export file in Q20 BYOB. The export contains timestamped RR intervals. The system matches the Garmin timestamp window (±30 seconds) against the stored stimulus_id captured_at time. If a Garmin reading exists within that window — it gets linked. If not — the anchor is confirmed from video + voice alone and marked as hardware_unconfirmed until the next upload.
