CHEF RECIPE FLOW — ARCHITECTURE DOCTRINE · PRE-BUILD LOCK
ALLERGY LAYER RULE
Every ingredient in every Chef suggestion runs through The Equalizer against the member's membrane. Not just the scanned item — every item in the full recipe list.
UPGRADE GATE
Non-full members see WITHOUT clearly, WITH blurred. Gate shows what they're missing. One CTA: Upgrade. No shame. No repeated asks.
SECOND CLAUDE CALL
Tapping a recipe chip fires a new Claude API call with: recipe name, member membrane, scanned items. Returns: full ingredient list, HAVE/FLAGGED/NEED split, allergy warnings, substitutes, prep note.
VERDICT LANGUAGE
"TAKE NOTICE" replaces "HEADS UP" permanently. Calm authority. Not alarm. Not dismissal. Notice it. Act on it.
SHOPPING MEMO STATES
DONE (green check), FLAGGED (red warning + allergy note + safe alternates + SCAN TO VERIFY), SCAN NOW (blue bolt), OPEN (empty checkbox).
SHARED RECIPE RULE
When shared via Live Cooking, the shared version shows the clean ingredient list only. Membrane flags and personal substitutes are never shared. Personal truth stays private.
COOKBOOK TABLE NEEDED
New Supabase table: cookbook_recipes. Fields: member_id, recipe_name, ingredients (jsonb), membrane_flags (jsonb), created_at, shared (boolean), live_cooking_session_id.
ROLE SEPARATION
The Chef suggests, builds lists, delivers prep notes, assembles cookbook. The Equalizer runs every allergy check. Act Right Dollars fire when a safe substitute saves money vs the flagged product.
